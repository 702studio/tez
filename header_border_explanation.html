Understanding How Borders Can Be "Removed" with CSS

Method 1: Standard border Property

This box has a standard border-bottom property.

This is the most straightforward way to add a border. It can be removed by setting border-bottom: none; or border: none;.

Method 2: Using an ::after Pseudo-element

Sometimes, a visual "border" or line is created using a pseudo-element like ::after. This pseudo-element is then styled to look like a border.

This box's 'border' is created by an ::after pseudo-element with a height (e.g., height: 3px;).
This box also uses an ::after pseudo-element for its 'border', but its height is set to 0px, making the 'border' disappear.

Explanation for header::after { height: 0px; } in the Site's CSS

In the main site's CSS (index.html), the header on mobile screens previously had a visual line at the bottom. This line was not created by the standard border CSS property directly on the header element. Instead, it was created by an ::after pseudo-element associated with the header.

This header::after pseudo-element is styled with properties such as:

  • content: ''; (necessary for any pseudo-element to render)
  • position: absolute; (to place it relative to the header)
  • bottom: 0; (to position it at the bottom edge of the header)
  • left: -20px; and width: calc(100% + 40px); (to make it span the full viewport width, accounting for padding)
  • background-color: var(--border-color); (to give it color)
  • And crucially, height: 1px; (this gave the line its thickness)

The height property of the pseudo-element is what gave the line its visible thickness. When the CSS rule header::after { height: 0px; } is applied (specifically for mobile views in this case), it instructs the browser to render that pseudo-element with no height.

An element (or pseudo-element) that has no height, and no other properties that would create visible dimensions (like its own padding or border properties), becomes invisible. Since the background-color can only fill the area defined by its dimensions, a height of 0px means there's no area for the background color to be visible.

Therefore, if the ::after pseudo-element is itself acting as the border (as it does in the site's header), setting its height to 0px effectively makes that visual border disappear. This is the technique used to hide the header's bottom line on mobile screens in the updated CSS.

This method is different from directly setting border: none; on an element if that element's border had been applied using the standard border property. In this scenario, we are manipulating the dimensions of the pseudo-element that constitutes the border, effectively removing it from view.
