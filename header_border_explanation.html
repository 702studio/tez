Understanding Header Borders and ::after

Standard Bottom Border

This is a standard div with a border-bottom and some padding-bottom.

Standard Border 
.standard-border {
    border-bottom: 2px solid blue;
    padding-bottom: 10px;
}

Hiding Border with ::after

The ::after pseudo-element can be used to draw over an element's border area. If this pseudo-element is given a background color matching the parent and a height that covers the original border's area (including any padding where the border line is drawn), it can obscure the original border.

Covered Border 
.border-covered-by-after {
    border-bottom: 2px solid red; /* Original border */
    padding-bottom: 10px;
}
.border-covered-by-after::after {
    content: '';
    position: absolute;
    bottom: 0;
    left: 0;
    width: 100%;
    background-color: #f0f0f0; /* Parent's background */
    height: 12px; /* Covers 10px padding + 2px border */
}

::after with height: 0px

In the user's specific case, if header::after was styled with height: 0px; and it still affected a border, it implies a few scenarios:

  1. The ::after element *was* the border: The pseudo-element itself might have been styled to create the visual border line (e.g., using its own background-color and a small height, or a border-top property). Setting its height to 0px would effectively remove this pseudo-element-based border if its own border properties were also removed or relied on its height.
  2. The ::after element covered a border and height: 0px removed that covering aspect: While less direct, if the ::after element had other properties (like a thick border-top of its own that was meant to look like the header's border) and height: 0px was part of a set of changes (e.g. also setting border-top: none on the pseudo-element), then it would contribute to removing the visible border effect.

Example 1: Pseudo-element as the border

Here, the ::after element creates a green line that acts as a border.

Pseudo Border 
.pseudo-border::after {
    content: '';
    position: absolute;
    bottom: -2px; /* Positioned like a border */
    left: 0;
    width: 100%;
    height: 2px; /* This IS the border */
    background-color: green;
}

If height: 0px; is applied to this ::after, the green "border" disappears:

Pseudo Border (height: 0px) 
.pseudo-border-removed::after {
    /* ... same positioning ... */
    height: 0px; /* Removes the pseudo-element "border" */
    background-color: green;
}

Example 2: Complex scenario (::after contributes to border effect)

Imagine the ::after had its own border-top and a background-color, contributing to a layered border effect. Setting its height to 0px could remove its background and, if its border-top was also removed or relied on the height, make the entire effect disappear. In the user's case, the header::after originally had height: 1px and a background-color, acting as a full-width line. Setting height: 0px directly removed this line.

Complex (height: 10px)

If its height is set to 0px (and assuming its border-top might also be removed or is ineffective at 0 height):

Complex (height: 0px) 
.complex-border-scenario::after {
    content: '';
    position: absolute;
    bottom: -10px;
    left: 0;
    width: 100%;
    height: 10px; /* Default height */
    border-top: 2px solid orange;
    background-color: lightyellow;
}
.complex-border-scenario.height-zero-effect::after {
    height: 0px; /* Removes background & potentially border-top's visibility */
    border-top: 2px solid orange; /* This border might become invisible or be removed by other styles */
}

User's specific case: The original header::after had height: 1px; background-color: var(--border-color);. This created a 1px line. Setting height: 0px; on this pseudo-element would directly remove that visible line because a background color cannot render in a 0px height area, and there was no separate border property on the pseudo-element itself.

General Explanation from Prompt

The ::after pseudo-element can be used to draw over or modify the appearance of an element's border area. In the "Hiding Border with ::after" example above, the first box has a standard bottom border. The second box has an ::after pseudo-element. This pseudo-element is given a background color and a height that covers the original border's area (including any padding where the border line is drawn), so it obscures the original border.

If header::after was styled with height: 0px; and it still removed a border, it implies that the border was not a simple border property on the header itself, but perhaps the ::after pseudo-element *was* the border (e.g. header::after { height: 1px; background-color: black; width: 100%; position: absolute; bottom: 0; left: 0; }). Setting its height to 0px would then effectively remove this pseudo-element-based border because an element with no height and no explicit border of its own won't be visible.

Alternatively, if the ::after pseudo-element had, for instance, a border-top: 1px solid red; and this was the intended "border" of the header, then setting height: 0px; on the pseudo-element *along with* changing its border-top to none (or a transparent color) would remove that border. The key is that height: 0px on a pseudo-element that either *constitutes* the border or is styled to cover an underlying border (and is then made transparent or its covering properties removed) can make a border effect disappear. In the user's original CSS, the header::after was a simple line created by its height and background-color, so setting height:0px was sufficient to remove it.
